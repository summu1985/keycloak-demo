Check the resources into GitHub

This step is required because the OpenShift BuildConfig builds the custom Keycloak image directly from your GitHub repository.

Copy the package contents into your repository:

git clone https://github.com/summu1985/keycloak-demo.git
cd keycloak-demo

cp -R /path/to/keycloak-demo-updated/* .

Commit and push:

git add .
git commit -m "Add reusable Keycloak custom authentication demo"
git push origin main

Confirm that these paths exist on GitHub:

extensions/Containerfile

Delete an earlier demo installation only when you no longer need its data:

oc delete project keycloak-demo

Wait until it is removed:

oc get project keycloak-demo

Then run the initial deployment:

./scripts/deploy.sh

oc get csv -n keycloak-demo

oc logs deployment/openldap -n keycloak-demo

oc rollout status deployment/openldap -n keycloak-demo

Expected:

deployment "openldap" successfully rolled out

Verify LDAP users

oc exec deployment/openldap \
  -n keycloak-demo \
  -- ldapsearch \
  -x \
  -H ldap://localhost:389 \
  -D "cn=admin,dc=demo,dc=local" \
  -w adminpass \
  -b "ou=people,dc=demo,dc=local" \
  "(objectClass=inetOrgPerson)" \
  uid cn mail

Expected response :

# extended LDIF
#
# LDAPv3
# base <ou=people,dc=demo,dc=local> with scope subtree
# filter: (objectClass=inetOrgPerson)
# requesting: uid cn mail 
#

# sp001, people, demo.local
dn: uid=sp001,ou=people,dc=demo,dc=local
cn: SP Employee
uid: sp001
mail: sp001@demo.local

# p001, people, demo.local
dn: uid=p001,ou=people,dc=demo,dc=local
cn: P Employee
uid: p001
mail: p001@demo.local

# search result
search: 2
result: 0 Success

# numResponses: 3
# numEntries: 2

Check the enrichment API:

oc logs deployment/token-enrichment-api -n keycloak-demo

Test the enrichment API from inside the cluster:

oc run curl-test \
  -n keycloak-demo \
  --rm -it \
  --restart=Never \
  --image=curlimages/curl \
  -- \
  curl -s "http://token-enrichment-api:8080/enrich?username=sp001"

Expected response:

{
  "userId": "EMP-SP001",
  "userType": "employee",
  "roles": [
    "employee_user",
    "employee_portal_access"
  ]
}

Build the custom Keycloak image

The custom image contains:

Username-pattern backend guard
External enrichment protocol mapper
Comma-separated role mapper

Start the build:

oc start-build customer-keycloak \
  -n keycloak-demo \
  --follow

Watch it separately with:

oc get builds -n keycloak-demo

View build logs:

oc logs \
  -n keycloak-demo \
  build/customer-keycloak-1 \
  -f
